## 4. Модель данных (PostgreSQL)

### 4.1 Основные сущности

| Таблица | Описание |
|---------|----------|
| `users` | Пользователи (id, email, password_hash, name, created_at) |
| `categories` | Категории товаров (id, name, parent_id, sort_order) |
| `products` | Товары (id, category_id, name, description, price, unit, image_url, stock) |
| `orders` | Заказы (id, user_id, status, total, address, created_at) |
| `order_items` | Позиции заказа (order_id, product_id, quantity, price) |
| `favorites` | Избранное (user_id, product_id) |
| `push_subscriptions` | Подписки на уведомления (user_id, token/channel, type) |

### 4.2 Схема связей

```
users ──┬──< orders >── order_items >── products
        │
        └── favorites ──> products
        └── push_subscriptions

categories ──> products (many-to-one)
```

## 5. REST API (основные эндпоинты)

### 5.1 Аутентификация
| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/auth/register` | Регистрация |
| POST | `/api/auth/login` | Вход (JWT) |
| POST | `/api/auth/refresh` | Обновление токена |
| POST | `/api/auth/logout` | Выход (инвалидация) |

### 5.2 Каталог
| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/categories` | Список категорий |
| GET | `/api/products` | Товары (фильтр по категории, поиск) |
| GET | `/api/products/{id}` | Детали товара |

### 5.3 Заказы
| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/orders` | Создать заказ |
| GET | `/api/orders` | История заказов |
| GET | `/api/orders/{id}` | Детали заказа |
| PATCH | `/api/orders/{id}/status` | Обновить статус (опционально) |

### 5.4 Избранное
| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/favorites` | Список избранного |
| POST | `/api/favorites/{productId}` | Добавить в избранное |
| DELETE | `/api/favorites/{productId}` | Удалить из избранного |

### 5.5 Подписки на уведомления
| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/subscriptions` | Подписаться на push |

## 6. WebSocket

### 6.1 Назначение
- Рассылка уведомлений в реальном времени (статус заказа, акции)
- Подключение аутентифицируется JWT

### 6.2 Протокол сообщений (JSON)

**Клиент → Сервер:**
```json
{
  "type": "subscribe",
  "channels": ["orders", "promotions"]
}
```

**Сервер → Клиент:**
```json
{
  "type": "notification",
  "channel": "orders",
  "payload": {
    "orderId": 123,
    "status": "shipped"
  }
}
```

### 6.3 Каналы подписки
| Канал | Описание |
|-------|----------|
| `orders` | Изменение статуса заказов пользователя |
| `promotions` | Акции и скидки |

## 7. Безопасность

| Аспект | Реализация |
|--------|------------|
| Аутентификация | JWT (access + refresh tokens) |
| Хранение паролей | BCrypt (strength 12) |
| Транспорт | HTTPS/WSS обязательны |
| Защита от атак | Rate limiting, CORS, валидация входных данных |
| Токены | Короткий срок жизни access token, HttpOnly для refresh (если веб) |
| Секреты | Переменные окружения, не в коде |
